All monitoring data collected from your devices and monitors is stored in a database structure designed to ensure speed, integrity, and historical accuracy.

### What this means for you



| **Technical Feature** | **User Benefit** |
| --- | --- |
| **NoSQL (Non-Relational)** | Ensures **high speed** and **efficiency** in storing and retrieving large volumes of data (Big Data), allowing the system to track metrics from countless devices in real time. |
| **Time Series** | Data is stored with an **exact timestamp**, organized in chronological order. This optimizes access and analysis of **trends and patterns over time**, making it ideal for monitoring. |
| **Cannot be Modified/Deleted/Reset** | Data collected from monitors cannot be tampered with for security. Once a data point (event, metric) is recorded, it becomes **permanent**. This feature is important for **auditing, compliance, and IT forensics**, as it ensures the history has never been altered. |



In summary, the combination of these technologies ensures you have access to a **fast, reliable, and tamper-proof performance history**.
